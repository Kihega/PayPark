# Sprint 2 — Vehicle Plate Number Verification

**Duration**: Week 4–5 (10 working days)
**Goal**: Field officers can enter a vehicle plate number and instantly retrieve
owner details, or receive a clear invalid-plate error.

---

## Sprint Goal
> "An officer enters a plate number and sees the owner's name and phone number
> within 2 seconds — or gets a clear 'plate not found' error."

---

## User Stories

### US-201 | Vehicle Registry Data Model & Seeding
**As a** system,
**I want** a vehicle registry database seeded with test data,
**So that** plate lookup works end-to-end before the national API integration.

**Acceptance Criteria:**
- [ ] `Vehicle` model created in Django with all required fields
- [ ] At least 200 seed records with realistic Tanzanian plate numbers
- [ ] Django management command `seed_vehicles` populates the database
- [ ] Seeded data covers: valid plates, invalid/unregistered plates, various
      vehicle categories (private, commercial, government)

**Tasks:**
- [ ] Create `vehicles` Django app
- [ ] Define `Vehicle` model:
      `plate_number` (unique, indexed), `owner_name`, `owner_phone`,
      `owner_email` (nullable), `make`, `model`, `color`, `category`
      [PRIVATE|COMMERCIAL|GOVERNMENT], `region`, `is_valid`, `registered_at`
- [ ] Write `management/commands/seed_vehicles.py` using Faker
- [ ] Add vehicle categories and regions as Choice fields
- [ ] Create and run migrations

---

### US-202 | Plate Verification API Endpoint
**As a** field officer,
**I want** to look up a plate number through the app,
**So that** I can confirm the vehicle is registered and get owner contact details.

**Acceptance Criteria:**
- [ ] `GET /api/vehicles/lookup/?plate=ABC123` returns vehicle + owner data
- [ ] Response includes: owner_name, owner_phone, owner_email, make, model,
      color, category, is_valid
- [ ] Plate not found returns HTTP 404 with `{ "error": "plate_not_found" }`
- [ ] Invalid plate format (wrong length/characters) returns HTTP 400
- [ ] Lookup is case-insensitive (abc123 = ABC123)
- [ ] Every lookup is logged to `audit_log` with officer ID and timestamp
- [ ] Response time < 500ms (database query with index on plate_number)

**Tasks:**
- [ ] Create `VehicleLookupView` (ListAPIView or APIView)
- [ ] Add plate format validator (regex for Tanzanian plates: e.g. T xxx ABC)
- [ ] Normalize plate number (strip spaces, uppercase) before query
- [ ] Write audit log entry on every lookup attempt
- [ ] Add database index on `plate_number` field
- [ ] Write unit tests:
      — Valid plate → 200 + owner data
      — Unknown plate → 404
      — Bad format → 400
      — Case-insensitive match → 200
- [ ] Add endpoint to Postman collection

### US-203 | Plate Number Input Screen (Mobile)
**As a** field officer,
**I want** a clean screen where I can type a plate number and tap "Verify",
**So that** I can quickly check any vehicle that arrives at the parking area.

**Acceptance Criteria:**
- [ ] Main screen (home) has a prominent plate number input field
- [ ] Input auto-uppercases characters as typed
- [ ] Plate format hint shown below input (e.g., "Format: T 123 ABC")
- [ ] "Verify Plate" button triggers API lookup
- [ ] Loading spinner shown during API call
- [ ] On success: navigate to Vehicle Detail screen
- [ ] On "plate not found": red error banner "Vehicle not found in registry"
- [ ] On network error: "Check your connection" with retry button

**Tasks:**
- [ ] Create `app/(app)/index.tsx` (Home / Plate Entry screen)
- [ ] Build `PlateInput` component with auto-uppercase and format mask
- [ ] `useVehicleLookup` query hook (React Query)
- [ ] Loading and error states in UI
- [ ] Navigate to `app/(app)/vehicle/[plate].tsx` on success, passing owner data

- [ ] Government/INVALID vehicles visually distinguished (colored badge)

**Tasks:**

## Tanzanian Plate Number Format Reference

  Private vehicles   : T [3 digits] [3 letters]   — e.g. T 123 ABC
  Government vehicles: G [3 digits] [3 letters]   — e.g. G 001 XYZ
  Commercial/Trucks  : TZ [series]                — varies
  Diplomats          : CD [series]

  Validation regex (simplified): `^[A-Z]{1,3}\s?\d{2,4}\s?[A-Z]{2,3}$`

## Notes & Risks

| Risk                                         | Mitigation                                  |
|----------------------------------------------|---------------------------------------------|
| National vehicle DB not available in Phase 1 | Use seeded data; design for API swap later  |
| Officers entering plates with typos          | Format mask + clear hint on input screen    |
| Privacy concern: exposing owner phone        | Restrict endpoint to authenticated officers |
