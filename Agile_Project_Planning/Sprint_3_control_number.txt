# Sprint 3 — Control Number Generation & Expiry Logic

**Duration**: Week 6–7 (10 working days)
**Goal**: Officers can generate a unique billing control number for a verified vehicle.
The system enforces the 5-hour expiry rule — no duplicate active bills per vehicle.

---

## Sprint Goal
> "One tap after plate verification generates a control number, sends it to the
> owner, and prevents any duplicate from being issued within 5 hours."

---

## User Stories

### US-301 | Control Number Data Model
**As a** system,
**I want** a robust ControlNumber model that tracks all billing events,
**So that** every transaction is permanently auditable.

**Acceptance Criteria:**
- [ ] `ControlNumber` model stores all required billing fields
- [ ] `status` field transitions: ACTIVE → EXPIRED or PAID
- [ ] `expires_at` is always exactly 5 hours after `generated_at`
- [ ] `control_number` field is globally unique and human-readable
- [ ] Model has database indexes on: plate_number, status, expires_at

**Tasks:**
- [ ] Create `billing` Django app
- [ ] Define `ControlNumber` model:
      `control_number` (unique), `plate_number`, `vehicle` (FK),
      `officer` (FK), `location` (FK), `amount_due` (Decimal),
      `generated_at`, `expires_at`, `status` [ACTIVE|EXPIRED|PAID],
      `sms_sent` (bool), `email_sent` (bool), `notes`
- [ ] Control number format: `PKP-YYYYMMDD-XXXXXX` (date + 6 random alphanum)
- [ ] `save()` override: auto-set `expires_at = generated_at + timedelta(hours=5)`
- [ ] Create and run migrations

### US-302 | Control Number Generation API
**As a** field officer,
**I want** to generate a billing control number for a verified vehicle,
**So that** the car owner receives an official bill and I have a record.

**Acceptance Criteria:**
- [ ] `POST /api/billing/generate/` with `{ plate_number, location_id }`
- [ ] System checks for any ACTIVE control number for the plate first
- [ ] If active control number exists:
      → Return HTTP 409 with `{ "error": "active_bill_exists",
        "expires_at": "...", "control_number": "..." }`
- [ ] If no active control number (or all expired):
      → Create new ControlNumber, return HTTP 201 with full bill details
- [ ] Fee amount determined by location and vehicle category automatically
- [ ] All generation events logged to `audit_log`
- [ ] Response time < 1 second

**Tasks:**
- [ ] Create `GenerateControlNumberView` (CreateAPIView)
- [ ] Write `check_active_control_number(plate_number)` service function
- [ ] Write `generate_control_number()` utility (PKP-YYYYMMDD-XXXXXX format)
- [ ] Write `calculate_fee(location, vehicle_category)` service
- [ ] Return 409 conflict with clear expiry info if duplicate found
- [ ] Write unit tests:
      — First bill for plate → 201 success
      — Second bill within 5hrs → 409 conflict
      — Second bill after 5hrs → 201 success (new bill)
      — Invalid plate_number → 400
      — Invalid location_id → 400
- [ ] Add endpoint to Postman collection

---

### US-303 | Automatic Expiry Background Task
**As a** system,
**I want** ACTIVE control numbers to be automatically marked EXPIRED
after 5 hours,
**So that** the database always reflects accurate status.

**Acceptance Criteria:**
- [ ] A background task runs every 15 minutes
- [ ] Task queries all ACTIVE control numbers where `expires_at <= now()`
- [ ] Updates their status to EXPIRED in a single bulk update
- [ ] Task execution is logged (count of records expired)
- [ ] Task failure does not crash the main API process

**Tasks:**
- [ ] Install and configure Celery with Redis broker
- [ ] Write Celery task `expire_control_numbers`
- [ ] Schedule task every 15 minutes using Celery Beat
- [ ] Write `docker-compose.yml` addition: Celery worker + Beat services
- [ ] Log task results to Django logger
- [ ] Write integration test: create ACTIVE bill with past expiry, run task,
      verify status = EXPIRED

## Control Number Format

  PKP - YYYYMMDD - XXXXXX
  │     │           └── 6 random alphanumeric (uppercase)
  │     └─────────────── Date of generation
  └───────────────────── ParkiPay prefix

  Example: PKP-20260513-A3K9P2

---

## Notes & Risks

| Risk                                       | Mitigation                                   |
|--------------------------------------------|----------------------------------------------|
| Redis unavailable → Celery tasks fail      | API still works; expiry checked on-demand    |
| Clock skew between servers                 | Use UTC everywhere; `django.utils.timezone`  |
| Duplicate control number collision         | UUID fallback + DB unique constraint         |
| Officer tries to bypass 5hr rule           | Server-side enforcement only; no client override |
