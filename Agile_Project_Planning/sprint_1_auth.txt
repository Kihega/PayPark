SPRINT GOAL
"A field officer can launch ParkiPay on their Android phone, log in with their
employee ID and password, see their profile, and log out securely. Tokens persist
across app restarts."

SUCCESS CRITERIA
  ✅ POST /api/auth/login/ returns access + refresh + officer profile
  ✅ POST /api/auth/refresh/ rotates the refresh token pair
  ✅ POST /api/auth/logout/ blacklists refresh token in DB
  ✅ GET /api/auth/me/ returns profile for valid token
  ✅ 5 failed logins lock account for 15 minutes
  ✅ Login screen — TATURA design, bilingual Swahili/English
  ✅ Tokens stored in expo-secure-store (hardware-backed)
  ✅ Silent 401 → refresh → retry via Axios interceptor
  ✅ Session persists across app restarts (SecureStore)
  ✅ Logout clears SecureStore + blacklists token
  ✅ All auth events written to audit_logs

USER STORIES
  US-101: Officer Login
    AC: Valid credentials → tokens + profile. Wrong password → error + remaining
        attempts. Missing fields → validation error.
  US-102: Persistent Session
    AC: SecureStore saves tokens. On relaunch → home screen directly.
        Splash shown while tokens load.
  US-103: Silent Token Refresh
    AC: 401 → Axios intercepts → calls /auth/refresh/ → retries → transparent.
  US-104: Account Lockout
    AC: 5 failed attempts → locked 15 min. Screen shows bilingual lockout message.
  US-105: Secure Logout
    AC: Confirmation dialog. POST /auth/logout/ blacklists token. SecureStore cleared.
  US-106: Login UI Design
    AC: TATURA green/yellow/black. Swahili + English. Keyboard avoidance. Eye toggle.

TECHNICAL NOTES
